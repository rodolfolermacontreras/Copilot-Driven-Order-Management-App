# System Architecture Guide

## Order Fulfillment Tracking System

---

### Overview

A Power Platform solution replacing manual Excel order tracking with a centralized,
AI-assisted order management system. Eliminates data loss and scaling issues
inherent in the previous Excel-based workflow.

---

### Components

#### 1. Data Layer

- **Dataverse Table: Orders**
  - Single table storing all order records
  - Seeded from Orders.xlsx (10 initial records)
  - Columns: Order Number, Product Name, Quantity, Order Date, Expected Delivery Date, Delivery Status

#### 2. User Interface Layer

- **Canvas App: Order Management App**
  - WelcomeScreen: Landing page with image, personalized greeting (User().FullName), navigation button
  - BrowseScreen: Gallery view with search, create new record icon, back icon
  - DetailScreen: Read-only view of a selected record (auto-generated by Power Apps)
  - EditScreen: Form for creating and editing records (auto-generated by Power Apps)

#### 3. Automation Layer

- **Power Automate Agent Flow: Create Order Flow**
  - Trigger: Called by Copilot Agent
  - Inputs: Product Name, Order Number, Quantity, Delivery Status
  - Action: Insert new row in Dataverse Orders table
  - Output: Confirmation text including the product name

#### 4. AI Layer

- **Copilot Agent: Order Record Assistant**
  - Platform: Copilot Studio
  - Topics: Create New Order (custom topic with trigger phrases)
  - Action: Calls Create Order Flow via agent action node
  - Response: Sends flow output message back to user
  - Channels: Microsoft Teams, Microsoft 365 Copilot Chat

---

### Data Flows

#### Order Creation via Copilot

```
User (Teams or Copilot Chat)
  --> Copilot Agent: Order Record Assistant
    --> Topic: Create New Order (trigger phrases detected)
      --> Action Node: Call Create Order Flow
        --> Power Automate: Create Order Flow
          --> Dataverse: INSERT new row in Orders table
        --> Flow Response: "Product [name] successfully added"
      --> Message Node: Send response to user
  --> User sees confirmation
```

#### Order Browsing via Canvas App

```
User
  --> Canvas App: WelcomeScreen
    --> Button: Navigate(BrowseScreen, ScreenTransition.Fade)
      --> BrowseScreen: Gallery of Orders (Dataverse)
        --> Search bar: Filter by field
        --> Select record --> DetailScreen (read-only)
        --> Create new icon --> EditScreen (form, new record)
        --> Edit selected --> EditScreen (form, existing record)
      --> Back icon: Navigate(WelcomeScreen, ScreenTransition.Back)
```
